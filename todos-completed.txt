TODO 1: SYSTEM RESOURCES COLLECTOR (HIGH PRIORITY) [COMPLETE]
================================================================================

WHY: Memory, CPU load, uptime, disk, and swap are fundamental monitoring metrics.
While node_exporter covers these on standard Linux hosts, OPNsense runs FreeBSD
and this data is available directly via the OPNsense API, making it valuable for
users who only run this exporter against their firewall.

API ENDPOINTS (all GET, all verified working):

1. GET api/diagnostics/system/systemResources
   Response:
   {
     "memory": {
       "total": "34157367296",
       "total_frmt": "32575",
       "used": 26293434401,
       "used_frmt": "25075",
       "arc": "9398875792",
       "arc_frmt": "8963",
       "arc_txt": "ARC size 8963 MB"
     }
   }

Note: Arc may be optional as not all systems will use zfs so we should not error if these do not exist 

2. GET api/diagnostics/system/systemTime
   Response:
   {
     "uptime": "23 days, 03:56:57",
     "datetime": "Sun Mar 1 17:12:35 GMT 2026",
     "boottime": "Fri Feb 6 13:15:38 GMT 2026",
     "config": "Thu Feb 26 14:03:49 GMT 2026",
     "loadavg": "0.43, 0.44, 0.46"
   }

3. GET api/diagnostics/system/systemDisk
   Response:
   {
     "devices": [
       {
         "device": "zroot/ROOT/default",
         "type": "zfs",
         "blocks": "876G",
         "used": "17G",
         "available": "859G",
         "used_pct": 2,
         "mountpoint": "/"
       }
     ]
   }

4. GET api/diagnostics/system/systemSwap
   Response:
   {
     "swap": [
       { "device": "/dev/ada0p3", "total": "8388608", "used": "223420" },
       { "device": "/dev/md98", "total": "2097152", "used": "219652" }
     ]
   }

PROPOSED METRICS:
  opnsense_system_memory_total_bytes        gauge   - memory.total (string→int)
  opnsense_system_memory_used_bytes         gauge   - memory.used (int)
  opnsense_system_memory_arc_bytes          gauge   - memory.arc (string→int)
  opnsense_system_uptime_seconds            gauge   - parse boottime to compute uptime
  opnsense_system_load_average              gauge   labels: [interval] - "1","5","15"
  opnsense_system_config_last_change        gauge   - parse config timestamp
  opnsense_system_disk_total_bytes          gauge   labels: [device, type, mountpoint]
  opnsense_system_disk_used_bytes           gauge   labels: [device, type, mountpoint]
  opnsense_system_disk_usage_ratio          gauge   labels: [device, type, mountpoint] - used_pct/100
  opnsense_system_swap_total_bytes          gauge   labels: [device]
  opnsense_system_swap_used_bytes           gauge   labels: [device]

IMPLEMENTATION:

Files to create/modify:
  - opnsense/system_resources.go  (new — Fetch method + structs)
  - internal/collector/system.go  (new — collector)
  - opnsense/client.go            (add 4 endpoints to map)
  - internal/collector/collector.go (add subsystem constant)
  - internal/options/collectors.go (add disable flag)

Add to opnsense/client.go endpoints map:
  "systemResources": "api/diagnostics/system/systemResources",
  "systemTime":      "api/diagnostics/system/systemTime",
  "systemDisk":      "api/diagnostics/system/systemDisk",
  "systemSwap":      "api/diagnostics/system/systemSwap",

Notes:
  - The boottime field is a human-readable string "Fri Feb 6 13:15:38 GMT 2026".
    Parse with time.Parse("Mon Jan 2 15:04:05 MST 2006", boottime) and compute
    uptime as time.Since(parsed).Seconds().
  - The loadavg field is comma-separated: split on ", " and parse each float.
  - Memory "total" and "arc" are strings, "used" is int — handle both types.
  - Disk "blocks"/"used"/"available" are human-readable (e.g. "876G") but used_pct
    is already numeric. For bytes, parse the human-readable strings or use used_pct.
    Actually simpler: just expose used_pct as a ratio and skip byte parsing for disks.
  - Swap total/used are in KB (strings).
  - Call all 4 endpoints in the single Update() method. This means 4 HTTP calls per
    scrape for this collector — acceptable since these are lightweight endpoints.
